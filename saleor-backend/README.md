# Self-hosted Saleor Backend

This folder is a Coolify-ready Saleor backend stack kept in the same repository as the storefront for easier development.

It uses the official open-source Saleor images:

- `ghcr.io/saleor/saleor:3.23`
- `ghcr.io/saleor/saleor-dashboard:3.23`
- `postgres:15-alpine`
- `valkey/valkey:8.1-alpine`
- `axllent/mailpit`

## Coolify

Deploy this folder as a separate Docker Compose application/resource with base directory:

```txt
/saleor-backend
```

`docker-compose.yaml` exposes:

- API URL: `http://saleor-api.159.69.35.245.sslip.io`
- Dashboard URL: `http://saleor-dashboard.159.69.35.245.sslip.io`
- Mailpit URL: `http://saleor-mailpit.159.69.35.245.sslip.io`

`docker-compose.yaml` uses Coolify magic variables to generate:

- Postgres credentials
- Saleor `SECRET_KEY`
- Initial admin password

The `saleor-init` service runs migrations and seeds sample data once. The initial admin email is:

```txt
admin@example.com
```

The password is generated by Coolify as `SERVICE_PASSWORD_ADMIN`.

## Notes

This is suitable for a first self-hosted backend deployment. Before production traffic, add durable backups, a real SMTP provider, object storage for media, domain names under the final site domain, monitoring, and payment apps.
